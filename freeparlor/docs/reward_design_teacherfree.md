## 1. 設計判断（3分岐、すべて A を採用）

### 分岐1: 共通通貨 = 点（千点単位）【採用】
素点・ウマオカ・チップを全て「点」で表現し同一通貨で合算。
円換算は学習に不要な定数倍、正規化は最適化側（Huber）の仕事であり報酬定義に混ぜない
（＝経済とスケールの分離）。現行 `calc_delta_blend` の単位と地続き。

### 分岐2: 着順は GRP 密化のまま維持【採用】
ウマオカは半荘末にしか確定しない疎な terminal reward。GRP で「今の局が最終着順期待を
どう動かしたか」に密化するのが着順の credit assignment を助ける実証済み装置。
収支一本化(B)だと着順報酬が半荘末1点に潰れ、序盤打牌に credit が届かない
（疎な terminal reward が序盤に届かない問題の再来）。GRP は教師依存が軽く、チップ盲も正しい。

### 分岐3: チップは単一 Q に MC 統合【採用】
後付け Q_chip ヘッド（per-move TD）は不活性で失敗が確定。撤去し、チップを素点と同じく
MC return に入れて単一の価値関数が「素点＋ウマオカ＋チップ」の総収支を予測する。
Phase5 の Q_chip / chip_target net / β_sel / chip_n_step は全廃 → 設計が大幅に simplify。

**※分岐3の留保（重要）:** 統合は「不活性な別ヘッドより素直」という理由であり、
**希少性は解決しない**（→ §4）。

## 2. 報酬式（本線ドラフト）

半荘の総収支（点）を最大化する単一価値関数。

```
per-kyoku reward =  α · (素点Δ / 1000)                 # 生MC・素点ストリーム
                 +  γ · (GRP着順期待ウマ の Δ)          # 密化・着順ストリーム（チップ盲=正しい）
                 +  β · (チップ枚数Δ × 5.0)             # 生MC・チップストリーム

Q が上記総和の MC return を学習（gamma^steps_to_done で割引）
最適化 = Huber(δ) で外れ値頑健化   # スケール問題はここで吸収（報酬定義には持ち込まない）
```

**撤去/不採用:**
- opp 項（λ_opp）: 廃棄（弱レバー・階層違い・守備歪めリスク）
- 別 Q_chip ヘッド / chip_target net / β_sel / chip_n_step: 全廃
- CQL: 教師と共に撤去（PPO 移行で。§3）

**係数の意味（確定）:**
- **α = γ = β = 1 が実収支と一致する経済定数。**
- スイープするのは「経済を変えたいとき」だけ（例: チップ 1000/3000 点版 = β or chip_value 変更）。
  学習安定のために係数を歪めるノブではない（歪めは Huber/正規化が担う）。

**鳴き頻度の決まり方（設計思想）:**
明示的な「鳴き項」は置かない。鳴き頻度は
(i) チップ経済が供給する鳴きインセンティブ（β=1 で赤鳴きが儲かる、Huber 検証で副露局収支が門前とパリティまで回復）
(ii) 鳴きの機会費用（守備低下・門前価値=立直/裏/一発の喪失）を単一 Q が実地で学ぶ
の綱引きで内生的に決まる —— という**予測**（要検証、§4）。
CQL(教師分布への引き戻し)で頻度を外から縛る現行方式を置き換える。

## 3. アルゴリズム移行（value-based → PPO）

現状は DQN（value-based）+ CQL。教師撤去で：
- **CQL は不要になる**: on-policy 自己対戦は全て試した行動＝OOD が原理的に無い。
  online 暴走(61%)の機構（OOD 過大評価）が構造ごと消える見込み。
- **鳴き頻度制御は報酬＋エントロピー正則化が担う**（教師非依存）。
- **スケール問題**: PPO では δ を選ぶ Huber より、**リターン正規化 / advantage 正規化**を
  設計に組み込む方が筋がいい（δ 選択不要）。ただし価値関数のスケールが他項と絡むため PPO 設計時に別途詰める。

**移行前に潰す予定の検証（GPU 順序）:**
1. （済）β=1 Huber offline 64×10 → スケール仮説確定。
2. （済 2026-07-02）192×40 Huber で崩壊回復を確定、小モデル交絡を排除（`beta1_huber_192x40_verify.md`）
3. PPO 移行設計（advantage 推定、エントロピー係数、self-play 相手プール、リターン正規化）。

## 4. 報酬式では解けない宿題（明記して先送り）

**希少性問題は本設計の射程外。** 赤鳴き正例 20件/28672局（0件ファイル率97.6%）は、
報酬をどう組んでも変わらない。単一 Q 統合は「不活性ヘッドよりマシ」なだけで希少性は未解決のまま。

- **Huber の守備範囲 = スケールのみ。希少性には一切効かない。**
- Huber 検証で鳴き和了率 1.61%→3.68% と倍増したのは追い風だが、人間 49%（副露和了）にはまだ遠い。
  **この gap は報酬式では埋まらない。**

希少性の根本解決は報酬式の外、PPO 移行以降で：
- **完全自己対戦で正例を自分で増やす**（教師を捨てる方向と直結）
- **探索**（エントロピー正則化等で赤鳴き試行を増やす）

→ (あ) の段階では「希少性は未解決の宿題」として杭を打ち、報酬式は「正しく値付けして単一 Q に入れる」
　までを完成とする。

## 6. sanity / 留意

- 本設計は紙上ドラフト。GPU 実験は §3 の順序で別途。
- 係数 α=γ=β=1 は「25000持ち/30000返し・ウマ ワンツー・1枚5000点」ルール基準の経済定数。
  対象ルールが変われば経済定数として再設定（歪めではない）。
- GRP 訓練データ源の教師依存度は未確認（`reward_audit_teacherfree.md` §5 の宿題）。
  PPO で GRP を self-play 再訓練するか現行流用かは移行設計時に判断。
- 希少性（§4）は報酬式の外の宿題として明示的に繰り延べ。
