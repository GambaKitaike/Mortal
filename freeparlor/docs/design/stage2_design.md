# Stage2 設計 — 赤濃縮カリキュラム（配牌 rejection sampling） (2026-07-06 確定版)

**位置づけ:** `ppo_migration_design.md` §5.2 の実装設計 + Stage1 結果
（`ppo_p3_stage1_result.md`）を受けた判定プロトコルの事前固定。
**判定条件は本書 commit 後に変更しない**（Stage1 と同じ pre-registration 規律）。

## 0. Stage1 からの入力（確定事実と解釈の限界）

- Stage1 判定: π(鳴き|可能∧赤保持) 窓平均 0.0581 = 初期比 **0.236×**、トレンド有意下降
  → 事前固定条件により Stage2 移行が確定
- 方策は「立直マキシマリズム」（π(立直|可能) 0.42→0.92）をチップ戦略として発見。
  立直局 +4.7 チップ/局 vs 鳴き局 −0.95。advantage も一貫して反鳴き
- **解釈の限界（重要）:** 確定したのは (a) 立直全ツは実在する強戦略（価値の下界）、
  (b) 自己対戦で安定な局所最適、(c) 純粋探索はそこから出ない — の3点まで。
  「鳴きが劣る」「立直が最適」は未証明（自己対戦の内輪であり搾取可能性は未検証、
  探索は計画された鳴き戦術を一度も試していない、初期化は門前偏重の事前分布、
  立直チップ報酬は稠密で赤鳴きチップは希少という学びやすさの非対称あり）

### 競合する2仮説（本 run はこの切り分け実験）

| 仮説 | 内容 | p_enrich の効き |
|---|---|---|
| **遭遇不足** | 赤鳴き→チップ和了の成功体験がサンプルされる機会自体が希少で、方策は鳴きの価値を判定できるデータを見ていない | **効く**（遭遇 約4倍で成功体験がサンプルされ始める） |
| **機会費用** | 方策は合理的に鳴きを拒否している。立直 +4.7チップという既知の太い戦略があるため、鳴き=確実なペイロードを捨てる行為として毎回罰される | **効かない**（遭遇が増えても同じ判断が繰り返される） |

Stage1 の結果は機会費用側の証拠を積んでいるが、Stage1 は遭遇**も**不足していたため
共犯の可能性が残る。単独犯の特定が本 run の目的（治療であり診断）。

## 1. Run 仕様（固定）

| 項目 | 値 |
|---|---|
| 開始 checkpoint | **init**（beta1_huber_192x40 — Stage1 と同一。16000 継続は不採用 → §7 Stage2b） |
| config | Stage1 と完全同一: lr=2e-5, τ=1.0, c_ent=0.01, eps_clip=0.2, ppo_epochs=4, 16,000 steps, save_every=2000 |
| 相手プール | Stage1 と同一構成（最新50%/過去K=5一様50%）、**init から共進化**（Stage1 の checkpoint は混ぜない） |
| 計装 | Stage1 と完全同一（action_mass / advantage_decomp / kyoku_reward_decomp / grp_calibration） |
| **変更点** | **p_enrich = 1.0 のみ。全 16k step で固定（run 中の変更禁止）** |

**init 採用の理由（2026-07-06 決定）:** Stage1 との差分を p_enrich 単独に保つ
（単一変数）。16000 継続は π(立直)=0.92 の凍結解除にエントロピー注射が必要となり
二変数化する。サンプル効率も init 有利（可塑性 × 遭遇4倍）。

## 2. 実装（rejection sampling）

- `board.rs` のシャッフル後、局ごとに確率 p_enrich で発動:
  trainee 席の配牌 13 枚に赤（5mr/5pr/5sr）が 1 枚以上含まれるまで再シャッフル
- 牌山 136 枚・赤 3 枚・点数計算・観測・mjai は**一切不変**
  （`akas_in_hand: [bool;3]` 前提を壊さない — 赤増加ルール案は棄却済み）
- p_enrich は config 値。**訓練 client のみ**に適用し、eval 経路は常時 p_enrich=0
  （構成 dump に p_enrich を含め、eval 側 0 を検定で assert）
- 自然分布での配牌赤≥1 は約 25%（実測: 検定(17)(c)、`_FirstLegalEngine` 自己対戦
  60 半荘・676 局、p_enrich=0.0、25.44%（172/676）。理論値
  1 − C(133,13)/C(136,13) ≈ 26.20% と整合。実装 2026-07-08）

## 3. 操作チェック（発進ゲート — 判定の前提条件）

- **step 500 時点で、鳴き可能局面のうち赤保持の割合 ≥ 54%**
  （Stage1 実測 36%（n=94,503/261,994）の 1.5 倍）
- 未達の場合は判定に進まず **run 停止 → 実装調査**（介入が効いていない）
- 検定 (17) として恒久化: 少数ゲームの self-play で配牌赤率が p_enrich=1.0 時に
  100%、p_enrich=0 時に自然率（~25%）であることを assert

## 4. 判定プロトコル（事前固定・変更禁止）

**主指標:** π(鳴き|可能∧赤保持)、n 加重。**窓:** step 8000–16000。
**ベースライン:** 当該 run の step 0–200。

| # | 条件 | 判定 |
|---|---|---|
| 1 | 窓平均が初期比 **≥ 2.0倍** | **遭遇不足仮説を支持（Stage2 成功）** → anneal 実験（§6）の設計に進む |
| 2 | 2.0倍未満 **かつ** 上昇トレンド無し（加重 OLS、Stage1 と同一手法） | **機会費用仮説を支持（Stage2 失敗）** → Stage3（構造化探索）解封の議論を解禁 |
| 3 | 倍率 [1.0, 2.0) だが slope/SE ≥ +3 かつ最終四半期単独 ≥2倍 | **上昇途中** → **+8000 step 延長を 1 回のみ**認め、延長後窓で 1/2 を再判定 |

**レンズ注意:** 本 run の測定は濃縮分布上。ベースラインも同一 run 内のため倍率は
内部整合するが、Stage1 の絶対値 (0.0581) との直接比較は不可。**Stage1 との比較は
倍率同士で行う**。

**成功時の副次確認（判定条件ではない、解釈材料として事前登録）:**
- kyoku_reward_decomp の鳴き局チップ平均が Stage1 の −0.95 から改善方向か
- π(立直|可能) の挙動（鳴きとの共存＝両取りが最強形の可能性を含めて観察）

## 5. eval バッテリー（3レンズ、run 完走後・GPU 1系統厳守）

1. **標準 argmax eval**: checkpoint 2000/4000/8000/12000/16000 + init、
   **自然分布（p_enrich=0）** 同一 seed 100 半荘。配備レンズ
2. **grp_baseline 対戦（1v3）**: DQN 時代の競技レンズ。Phase4 系数値と世代跨ぎ比較
   （エンジン互換は検定(9)で担保済み）
3. **メタ対決 probe**: Stage2 最終モデル 1 席 vs **Stage1 step16000 立直マシン 3 席**。
   濃縮で立った戦術（立ったなら）が立直メタ相手に生存するかの搾取可能性テスト第一歩

## 6. anneal（2b フェーズ）

本 run には含めない。**判定 1（成功）成立時のみ**設計する。固定するのはトリガーのみ:
成功 → 鳴きの立ち方（どの局面クラスで立ったか）を分析した上で p_enrich 1.0→0 の
降下スケジュールを設計し、自然分布での定着を別 run で検証する。
