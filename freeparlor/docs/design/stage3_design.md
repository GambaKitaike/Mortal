## 0. Stage2 からの入力（確定事実と本 run の標的）

- Stage2 判定（2026-07-11）: π(鳴き|可能∧赤保持) 窓平均 0.0485 = 初期比 **0.185×**
  （閾値 2.0×）、slope/SE=−2.56 → **分岐2成立、機会費用仮説を支持**。
  遭遇機会は設計通り 2.6 倍に増えていた（窓内 n=247,670 vs Stage1 94,503）のに
  試行率は Stage1（0.236×）より深く沈んだ — **遭遇不足の単独犯説は棄却**
- 7a（機会費用の機構）: 濃縮された赤供給は鳴きより先に立直を肥やし、機会費用ギャップ
  （立直局−鳴き局のチップ差）は 5.68 → 7.81 へ**拡大**。治療が病理を強化した
- 7c（**配備税**）: p_enrich=1.0 分布育ちの方策は自然分布への配備で有意な性能損失
  （合算 −8.65±3.87、avg_rank 2.475→2.705）。**新設計制約: 訓練分布・報酬への介入は
  anneal（正典への返却）とセットで設計する。anneal は成功時のオプションではなく必修**
- **残る交絡（本 run の標的）:** π(鳴き)≈0.05 まで沈んだ方策は、鳴いた後のプレイ
  （鳴き後の手組み・押し引き）が未熟なまま。Stage1/2 で測定された機会費用は
  「鳴きが割に合わない」と「この方策が鳴きを下手にしかプレイできない」を分離できていない
  （Stage1 §0 解釈限界「探索は計画された鳴き戦術を一度も試していない」がそのまま残存）

### 競合する2仮説（本 run はこの切り分け実験）

| 仮説 | 内容 | ボーナス+anneal の効き |
|---|---|---|
| **競技力ギャップ** | 機会費用は未熟さの見かけ。強制試行で鳴き局の競技力が形成されれば、鳴きは正典経済の下でも採用され続ける | **効く**（anneal 後も試行率が残留する） |
| **本質的機会費用** | 競技力が形成されても立直経路のペイロードが上回る。この経済・自己対戦設定では鳴きは本当に劣後 | **効かない**（anneal 後に逆戻りする） |

  ボーナスは**経済ではなく足場（scaffolding）**と定義する。run 内で b→0 へ anneal し、

  本 run は介入（報酬ボーナス）を run 内で正典に返却する設計をはじめから内蔵する。

## 2. Run 仕様（固定）

| 項目 | 値 |
|---|---|
| 開始 checkpoint | **init**（beta1_huber_192x40 — Stage1/2 と同一） |
| 相手プール | Stage1/2 と同一構成（最新50%/過去K=5一様50%）、**init から共進化**（Stage1/2 checkpoint 混入なし） |
| 配牌分布 | **p_enrich=0（自然分布、全経路）** |
| config | Stage1 と完全同一: lr=2e-5, τ=1.0, c_ent=0.01, eps_clip=0.2, ppo_epochs=4, 16,000 steps, save_every=2000 |
| 計装 | Stage2 と同一（action_mass / advantage_decomp / kyoku_reward_decomp / grp_calibration）+ ボーナス分離ログ（下記） |
| **変更点** | **鳴きボーナススケジュール1変数のみ**（1 branch = 1 variable） |

### ボーナス定義

- 訓練 client の trainee 席で、**鳴き可能∧赤保持の局面で鳴きを実行した decision step の
  報酬に b を加算**（per-decision 配置。局末の at_kyoku 配置とは独立の加算）
- 「鳴き可能」「赤保持」「鳴き実行」の定義は既存計装（π(鳴き|可能∧赤保持) の
  n_call_possible 系カウンタ）と**同一**。新しい条件定義を作らない
- 単位は千点（正典報酬と同一通貨）。b=5.0 は 1 チップ相当/鳴き実行

### スケジュール（事前固定・run 中の変更禁止）

| 区間 | b |
|---|---|
| step 0–4000 | **5.0 一定** |
| step 4000–8000 | **線形 anneal 5.0 → 0** |
| step 8000–16000 | **0（正典報酬のみ。判定窓はここ）** |

### 適用範囲・分離ログ

- ボーナスは**訓練 client のみ**。eval 経路は常時 b=0
  （p_enrich と同じ配線パターン: 構成 dump にボーナス値を含め、eval 側 0 を検定で assert）
- ボーナスは kyoku_reward_decomp の正典3ストリーム（素点/GRP/チップ）に**混ぜず、
  第4ストリームとして分離ログ**する。判定・副次確認はすべて正典3ストリームで行う

## 3. 発進ゲート（操作チェック — 判定の前提条件）

- **step 500 時点で、sampled π(鳴き|可能∧赤保持) ≥ 2.0 × 当該 run ベースライン
  （step 0–200）**

- 較正チェック: ベースラインは Stage1 の step 0–200 実測 **0.2462** と整合するはず
  （同一 init・同一自然分布・同一測定）。有意に乖離したら配管疑いとして停止・調査

## 4. 判定プロトコル（事前固定・変更禁止）

**主指標:** π(鳴き|可能∧赤保持)、n 加重。**窓:** step 8000–16000（= anneal 完了後、
正典報酬のみの区間）。**ベースライン:** 当該 run の step 0–200。

**レンズ注意:** 本 run の訓練測定は**自然分布上**。Stage1 と絶対値の直接比較が復活する
（Stage2 の濃縮分布測定とは倍率同士でのみ比較 — 従来規律どおり）。

| # | 条件 | 判定 |
|---|---|---|
| 1 | 窓平均が初期比 **≥ 2.0倍** かつ有意下降トレンド無し（加重 OLS、Stage1/2 と同一手法） | **競技力ギャップ仮説を支持（Stage3 成功）** → 配備検証（§6 レンズ2 の配備税チェック）と Stage2b 再設計に進む |
| 2 | 窓平均 **< 1.0倍**、または有意下降トレンドで減衰中 | **本質的機会費用仮説を支持（Stage3 失敗）** → 探索ラダー閉幕。「この経済・自己対戦設定では立直マキシマリズムが強制試行後も生存する」を商用知見として確定 |
| 3 | 倍率 [1.0, 2.0) かつ有意下降トレンド無し | **残留途中/部分残留** → **+8000 step 延長を 1 回のみ**認め、延長後窓（16000–24000、b=0 のまま）で 1/2 を再判定 |

**副次確認（判定条件ではない、解釈材料として事前登録）:**
- ボーナス期（step 0–8000）に鳴き局の**正典3ストリーム収支**（ボーナス除外）が
  Stage1 の −0.95（チップ/局）から改善方向か = **競技力形成の実測**。
  これが立たないまま分岐2（逆戻り）に至った場合、解釈は「機会費用が本質」ではなく
  「ボーナス期でも競技力を形成できず」に**弱まる**（本設計の反証条件）
- π(立直|可能) の挙動（鳴きとの共存 = 両取り形の可能性を含めて観察）
- advantage_decomp の call_taken vs call_declined ギャップの推移
  （anneal 前後で符号・大きさがどう動くか）

## 5. 監視期待値の事前書き換え

- **ボーナス期（step 0–8000）の副露率スパイクは意図された強制探索であり異常ではない**。
  argmax eval・自己対戦統計の副露率が Phase 系の健全帯を逸脱しても、ボーナス期は
  インシデント扱いしない（偽インシデント防止のため事前に明記）
- anneal 完了直後の critic explained variance の低下・過渡は既知クラスとして記録のみ
  （critic はボーナス込みリターンを一時学習しており、unlearning の過渡が予期される）
- 必須監視項目は従来どおり: `trajectory step count mismatch` = 0、
  `illegal_action_fallback_count` = 0、`online chip resolution failed` = 0、
  trainer NaN = 0、alive clients、step 到達性。凍結ルールも従来どおり
  （発進後は step 16000 完走までコード・config 変更禁止、例外はクラッシュと
  データ整合性の破れのみ）

## 6. eval バッテリー（run 完走後・GPU 1系統厳守・3レンズ）

1. **標準 argmax eval**: checkpoint 2000/4000/8000/12000/16000 + init、自然分布・
   同一 seed 100 半荘。構成 dump で **b=0 と p_enrich=0 を assert**
2. **grp_baseline 1v3**（配備レンズ）: Stage1/2 と同一固定 baseline
   （beta1_huber_192x40/mortal.pth。grp_baseline.pth は配管 fixture のため使用禁止）。
   **配備税チェックを事前登録:** step16000 vs init の各ストリーム差を 2SE 基準で判定
   （Stage2 7c と同一手法・同一 seed 系）。分岐1成立時はここが配備可否の関門
3. **メタ対決 probe**: Stage3 step16000（1席）vs **Stage1 step16000 立直マシン（3席）**。
   鳴き競技力を持つ方策が立直全ツ均衡を**搾取できるか**の初テスト
   （既存 `eval_meta_stage1_vs_stage2.py` ハーネスの challenger 差し替えで実施可能）
