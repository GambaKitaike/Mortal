## 0. 目的 — 競合仮説の直接切り分け

Stage1 以来残存する問い「この方策が鳴かないのはなぜか」の候補:

| # | 仮説 | DRCA での署名 |
|---|---|---|
| (i) | **内在的機会費用** — この経済・自己対戦設定では鳴きは本当に損 | ΔQ̄ が有意に負（鳴きスキルの高い方策でも負のまま） |
| (ii) | **credit assignment 失敗** — 鳴きは得なのに単一軌跡 GAE では推定できていない | ΔQ̄ が有意に正なのに当該方策の π(鳴き) は低迷 |
| (iii) | **競技力ギャップ** — 鳴き後の打ち回しが未熟だから損に見える | 共通局面セット上で、鳴きスキルの高い checkpoint ほど ΔQ̄ が正側へ動く |
| (iv) | **報酬設計の誤指定**（本プローブで新たに検出可能になる第四仮説） | 訓練合成報酬レンズで ΔQ̄ 負 × 実順位レンズで Δ 正（またはその逆） |

理論的位置づけ: RLOO（leave-one-out baseline）・COMA（counterfactual baseline）系の
分散削減手法のバリエーション。麻雀では山が配牌時に固定された物理的実在であるため、
同一山ペア差分は分散削減トリックではなく**当該配牌条件下の因果効果の不偏推定**そのもの
（duplicate 橋・同卓再現方式と同じ発想）。鳴きによるツモ順の乖離はノイズではなく
鳴きの帰結の一部であり、ペア差分に正しく含まれる。

### Stage3 判定結果との接続（どちらに転んでも情報を出す）

| Stage3 判定 | 本プローブが埋める穴 |
|---|---|
| 分岐1（残留 = 競技力ギャップ支持） | 残留した鳴き行動の**経済的裏付け**を定量化（π が残っても ΔQ̄ が負なら anneal 後の再崩壊予備軍） |

## 1. 手法定義

- **分岐点 s**: 鳴き可能∧赤保持の decision step。定義は既存計装
  （n_call_possible 系カウンタ、Stage3 の sel 定義）と**同一**。新しい条件定義を作らない
- **call 腕**: 分岐点で鳴き宣言を強制。鳴き選択肢が複数ある場合（チーの構成牌選択等）は
  **π を鳴き行動群に制限して再正規化サンプリング**（方策の鳴き方選好を保存）。
  鳴き実行後の打牌以降は全席とも素の π サンプリング
- **no-call 腕**: 分岐点で **π を非鳴き行動群（合法 mask ∧ ¬鳴き行動群）に制限して
  再正規化サンプリング**（call 腕と対称の操作化）。以降（同巡の他家の鳴き含め）全席フリー。
  **【2026-07-13 amendment（測定開始前・Gamba 裁定）】** 原文は「当該鳴き機会のパスのみ
  強制」だったが、監督側の実装検証スモークで**自分の手番の暗槓/加槓機会**（凍結済み
  分岐点定義 sel = 鳴き可能∧赤保持 が正当に含むクラス）ではパス行動（45）が mask 上
  存在せず定義不能であることが判明（実測: `call_types_available=[42]`・元の行動=打牌の
  分岐点で assert 発火）。非鳴き制限サンプリングは (i) パスが唯一の非鳴き行動である
  通常のリアクション局面では従来定義と同一挙動に帰着し、(ii) 手番中の槓機会では
  「槓せず π どおり打牌/立直等」、(iii) ロン可能なリアクションではロンを許容（鳴きの
  機会費用にロン見逃しを含めない）となる。分岐点定義（§5a-4）・解釈条件（§5a）は不変
- **ロールアウト**: 各腕 K 本（初期値 K=8、パイロットで腕内分散を見て確定）、
  局終了まで。腕内の K 本はサンプリング RNG のみ独立
- **推定量**: ΔQ̂(s) = mean(R_call) − mean(R_nocall)。単位は千点

### 報酬レンズ（2本、主・副）

| レンズ | 定義 | 答える問い |
|---|---|---|
| **主: 訓練合成報酬** | 正典3ストリーム合成（α素点+γGRP+βチップ、α=β=γ=1）を訓練時と同一の局終了配置で算出 | 「**訓練信号は**鳴きを何と言っているか」 |
| **副: 実順位** | 半荘完走し最終順位・最終スコア（ウマオカ+チップ込み）で測定。縮小 N | 「鳴きは**実際に**得か」 |

2レンズの符号が割れた場合が仮説 (iv)（報酬設計誤指定）の検出条件。
主レンズのみなら局終了まで、副レンズは半荘完走が必要なためコストが大きい（§6）。

## 2. 実装方式: fork-by-replay（libriichi 変更ゼロ）

mid-game の Board/Game deep clone（フォーク API = 新規 PyO3 表面）は**不要**。
以下の決定論性を利用して「同一局の再走 + 台本再生 + 分岐点強制」で分岐を実現する:

1. **山は seed 決定論的**: `OneVsThree::py_vs_py` 系は per-game seed を受け取り、
   RNG（ChaCha12Rng）は山シャッフル直後に破棄され構造体に保持されない
   （`board.rs` 確認済み 2026-07-12）。同一 seed → 同一山・同一配牌
2. **行動決定は Python 側コールバックに完全委任**: `react_batch`
   （`mortal/ppo_engine.py:105`）が全 decision を返す。**台本再生ラッパーエンジン**
   （採取済みログの行動列を分岐点まで4席全員分そのまま返し、分岐点で腕の強制行動を
   返し、以降は素のエンジンへ委譲）は純 Python で書ける
3. **台本再生区間は NN 推論不要**（ログの行動を返すだけ）。実コストは分岐後の
   並走ロールアウト（2 × K 本 × 局残り）のみで、これは手法上不可避な分

### 分岐点の採取

被測定 checkpoint による self-play を**純 π サンプリング**（訓練 client と同一構成 —
検定・診断 client の同一構成規律に準拠）+ 自然分布（p_enrich=0）で走らせ、
イベントログ + seed + decision index を記録。鳴き可能∧赤保持の decision step から
N 点を無作為抽出する。

### 規律との整合（明文化必須）

- **「訓練 rollout への行動上書き禁止」との分界線**: 本プローブは eval 側の反実仮想
  測定であり、行動強制は手法の定義そのもの。**訓練経路（client.py rollout・学習ループ）
  には一切触れない**。生成データは学習に流用しない
- eval 規約の継承: 全経路で **p_enrich=0 / call_bonus_b=0 を構成 dump に含め assert**
- **サイレントフォールバック禁止の適用**: 台本再生 prefix は分岐点到達時に
  **再現イベント列のハッシュ一致を assert**（不一致 = 決定論性の破れ = 大声で FAIL。
  黙って再試行しない）。call 腕の強制行動は合法手 mask 内であることを assert

### 実装時に検証すべきリスク（本書時点では grep レベルの確認のみ）

- (r1) `react_batch` のバッチ内 game 対応付け（`step_meta` 経由の識別で台本再生対象
  game を特定できるか）。不可なら batch=1 直列で逃げられる（遅いが診断用途には十分）
- (r2) 山生成以外に seed 依存の RNG 消費源がないこと（prefix 再現ハッシュ assert が
  そのまま検定になる）
- (r3) 鳴き機会が「他家打牌への反応」として全席に同時提示されるケースの台本再生順序

## 3. 測定計画

### 対象 checkpoint

| checkpoint | 役割 |
|---|---|
| init（beta1_huber_192x40） | 基準点（較正済みベースライン π≈0.25–0.27） |
| Stage1 final（step 16000） | 立直マキシマリスト（鳴きスキル最低群） |
| Stage3 step 16000 | 強制試行で鳴きスキルを形成した方策（判定結果により解釈が変わる） |
| （任意）Stage3 step 8000 | anneal 完了直後（余力があれば） |

### 分岐点セット（2系統併用）

- **(a) 各方策固有セット（主・生態学的レンズ）**: 各 checkpoint 自身の self-play から
  採取した局面で、自身の続行により測定（全席 = 被検方策の自己対戦）。
  「その方策が実際に直面する局面で、その方策にとって鳴きは得か」
- **(b) 共通参照セット（競技力ギャップレンズ）**: 参照方策（**Stage1 final** = 現行
  均衡の代表）の self-play から採取した固定局面セット。続行は **trainee 席のみ
  被検 checkpoint に差し替え、他3席は参照方策に固定**（相手スキルを固定して
  続行スキルの寄与だけを分離）。同一局面 × 同一相手で checkpoint 間のペア比較が成立し、
  仮説 (iii) の最直接検定になる

### 規模と統計

- N ≈ 500 分岐点 / セット、K = 8 / 腕（パイロット ~50 分岐点で分散を見て確定）
- 副レンズ（実順位・半荘完走）は縮小 N ≈ 100–150
- **主指標**: ΔQ̂ の n 加重平均と SE（**分岐点単位の cluster-robust**。同一分岐点の
  2K 本は独立でない）。符号検定を併記
- **層別分析（テンパイ距離・局進行度・点況・鳴き種別）は exploratory と事前明記**。
  主判定は全体平均のみ（post-hoc goalpost 防止）

### 5a-1. 規模（暫定値 + パイロットによる確定規則 — 規則自体を事前登録）

- **暫定**: K=8 本/腕、主レンズ N=500 分岐点/測定枠、副レンズ N=120（セット(a) のみ）
- **パイロット**: セット(a) × Stage1-16000 × 主レンズで先頭 50 分岐点（K=8）。
  パイロット分岐点は本測定に合算する（同一プロトコル・同一 seed 系列の先頭区間であり
  別実験ではない）
- **確定規則（唯一の許容 amendment はこの規則の機械的適用）**:
  - **N**: パイロットの分岐点単位 cluster SE から、ΔQ̄ の 95%CI 半幅 ≤ 1.0 千点を
    満たす N を外挿。下限 300・上限 800 でクリップ
  - **K**: パイロット K=8 の各腕を無作為に半分へ間引いた K=4 相当と比較し、
    全体 SE の劣化が 10% 未満なら K=4、以上なら K=8 を採用（K の増額はしない）
  - **壁時計超過時の削減順**: パイロット実測からの全測定枠projection が GPU 48h を

    Stage1-16000 / Stage3-16000 の2枠に絞る、の順で削減（この順以外の削減は不可）

### 5a-2. 閾値・主 contrast（事前列挙 — これ以外は全て exploratory）

- **有意基準**: |ΔQ̄| ≥ 2 × SE（分岐点単位の cluster-robust SE）。多重比較補正なし

- **主 contrast リスト**:
  1. セット(a) × 主レンズの ΔQ̄ — 4 checkpoint で計 4 本（解釈 A/B の判定材料）
  2. セット(b) 上の checkpoint 間ペア差 — init vs Stage1-16000、
     Stage3-16000 vs Stage1-16000 の 2 本（解釈 C: 鳴きスキル軸の正側移動）
  3. レンズ間符号割れ検定 — 副レンズ対象 checkpoint（セット(a) の 4 枠）ごとに
     主・副両レンズが**それぞれ 2SE 有意かつ逆符号**の場合のみ解釈 D を発動
- **解釈 B の π 低迷側の基準**: 当該 checkpoint の π(鳴き|可能∧赤保持) 実測
  （採取 self-play 中の n 加重平均）が較正バンド上限 0.31 未満であること
  （「低迷」の定義を post-hoc にしない）

## 6. コスト見積り（オーダー）

- 主レンズ: 2腕 × K=8 × N=500 = 8,000 局分の**後半戦**ロールアウト / (checkpoint × セット)。

  （平均分岐位置が局中盤なら実効 ~20,000 局相当）
- 副レンズ: 縮小 N で半荘完走。主レンズの 1–2 割増し
- prefix は推論ゼロのためロールアウト支配。既存 eval バッテリー数本分のオーダーで、
  GPU 1系統・直列実行の枠内。正確な壁時計はパイロットで実測してから本測定規模を確定
- ディスク: 生成ログは診断用途のみ。run 完走後の drain 清掃運用（バックログ7）に準じ、
  集計後に生ログ削除可（集計スクリプトと要約のみ commit）
