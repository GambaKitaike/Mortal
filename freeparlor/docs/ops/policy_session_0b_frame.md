## 0. セッションで決めること（3+1 議題）

1. **立直マキシマリズムの商用採否** — 現行均衡（鳴き最小・立直特化）の方策を
   フリー雀荘 AI として採用するか
2. **経済定数変更** — ウマオカ・チップ単価・β 等を動かす新実験系を開くか
3. **敵対的搾取者訓練の要否** — 現行均衡を搾取する方策を訓練して頑健性を検査するか
4. （付随）**Stage2b 解凍実験の再評価**（バックログ3） — 配備税の発見で価値が
   上がった「収束済み方策の分布シフト適応」

## 1. 確定済みの事実ベース（DRCA 結果に依存しない）

- **探索ラダー3段全て不成立**: Stage1 純探索（0.236×）→ Stage2 分布介入（0.185× +
  配備税）→ Stage3 報酬介入（anneal 後 1.066×・有意減衰）。立直マキシマリズムは
  経路によらない頑健な均衡
- **正典反鳴き勾配は定量再現**: 判定窓 raw advantage は call_taken +0.018 vs
  declined +0.522（Stage3）≈ Stage1（+0.012 / +0.524）
- **機会費用ギャップ ~5.5–5.9 チップが支配項**。副次確認: 鳴き局正典収支は改善方向
  （チップ/局 −0.95 → −0.63 → −0.33）だが損益分岐に届かず
- **配備税は介入形態依存**: 分布介入（Stage2）は有意、anneal 内蔵の報酬介入
  （Stage3）はゼロ → 「介入は anneal とセット」制約は対照実験済み
- **メタ対決**: 残留鳴きは立直メタで負債（Stage3-16000 vs Stage1-16000 チップ
  −0.675±0.236 ≈ −2.9SE）
- **DRCA パイロット**: ΔQ̂ = −0.4575 ± 1.5882 千点（n=50、n.s.）。本測定 N=485 で
  95%CI 半幅 ≤1.0 千点まで絞られる
- **§5a-1b 適用済み**: セット(b)×init 脱落。解釈 C は
  「Stage3-16000 vs Stage1-16000（セット(b)）」の1本で判定

## 2. DRCA 判定 × 議題の分岐（討議用シナリオ）

主判定はセット(a)×主レンズの ΔQ̄（4 checkpoint）+ contrast 2/3（§5a-2）。
解釈 A–D の条件文言は `drca_probe_design.md` §4（凍結）が正。

### シナリオ A — 内在的機会費用（全 checkpoint で ΔQ̄ < 0 有意、(b) で正側移動なし）

- 技術的含意: 方策は正しく損を学んでいる。phase 2（DRCA 訓練組み込み）は
  正当化されない（§4 A の凍結文言どおり）
- 議題1: 立直マキシマリズムは**当該経済設定での正解**。採否は「鳴かない瑕疵」では
  なく絶対競技力・商品性で判断する土俵に移る
- 議題2: 「鳴きのある AI を見たい」なら**経済定数変更が唯一の残存経路**
  （感度実験系の設計へ）
- 議題3: 独立に価値あり（均衡の頑健性検査として）

### シナリオ B — credit assignment 失敗（ΔQ̄ > 0 有意 かつ π(鳴き|可能∧赤保持) < 0.31）

- 技術的含意: 学習器が「得」を見えていない。phase 2 の検討解禁（§7）
- 議題1: 商用化は保留寄り（伸び代が残っている）。phase 2 の成否を先に見る選択肢
- 議題2: 定数変更の前に学習器側の修理が先
- 議題3: 価値上昇（搾取可能性が実在する示唆）

### シナリオ C — 競技力ギャップ（(b) で Stage3-16000 が Stage1-16000 より有意に正側）

- 技術的含意: 鳴きスキルを付ければ得になる。**C∧B は最高介入価値の象限**（§4）
- 議題1/2/3: B に準じるが、介入手段はスキル訓練（カリキュラム等）が第一候補になる

### シナリオ D — レンズ間符号割れ（主・副レンズが両方有意で逆符号）

- 技術的含意: 報酬設計誤指定の疑い → `reward_design_teacherfree.md` 再訪を起票
  （§4 D）。**議題1–3 は全て凍結し再監査を優先**

### シナリオ N — 全て n.s.（0 近傍・検出力は充足）

- 技術的含意: 赤保持鳴きの機会費用 ≈ 0（鳴いても鳴かなくても大差ない）
- 議題1: 「鳴かない損」も「鳴く得」もない → 純粋に商品性（見た目の打牌选択）の
  問題に還元。ε-greedy 的な演出混合等の非学習的解も土俵に入る
- 議題2/3: 動機は弱まる

## 5. 商品性要件ヒアリング結果（2026-07-24、Gamba 回答・裁定非関与）

§3 の最大の未充足だった「商品性の要件定義」を Gamba ヒアリングで埋めた記録。
これは要件の**記録**であって議題1–4 の裁定ではない（裁定は DRCA 判定後）。

### 5.1 確定した6軸

| 軸 | Gamba 回答 |
|---|---|
| 商品形態 | **検討/サジェストツール**（人間が打つ手を助言。対戦相手ではない） |
| 最優先の性質 | **絶対的な強さ**（打ち筋は結果が伴えば不問） |
| 「ほぼ鳴かない立直特化」の許容 | **許容できる**（強ければそのまま採用検討に進む） |
| 強さの合否基準 | **人間（Gamba/専門家）との実戦**（収支・打牌妥当性の主観判定） |
| サジェスト出力形式 | **上位N手 + 評価値**（複数候補を評価値付きで提示） |
| 経済プリセット軸 | **鳴き祝儀/面前祝儀の違い・ウマオカ表・チップ**（複数選択） |

### 5.2 §2 シナリオ・議題への含意

1. **商品形態がサジェストツール → §2 が暗黙に置いていた「対戦相手（打ち筋を人に見られる）」
   前提が外れる。** 重要なのは "AI がどう打つか" ではなく "人間に何を薦めるか"。
   シナリオ N の「ε-greedy 的演出混合」やメタ対決レンズ（7d）の商品的重みは低下する
   （見た目の自然さではなく助言の正しさが評価軸）。
2. **強さ最優先 + 立直特化許容 → 議題1 の「鳴かない瑕疵」論点は消滅。** DRCA が
   シナリオ A（内在的機会費用）または N（機会費用≈0）を出せば、商用採否は
   純粋に "強さ（人間実戦での合否）" の一点に収束する。逆に B/C なら伸び代が助言精度に
   直結するため phase 2 検討の動機が商品面からも立つ。
3. **DRCA が診断計測器から "商品の中核クレーム" に格上げされる。** サジェストツールが
   「鳴くな」と助言するとき、その正しさは ΔQ̂（鳴きの反実仮想価値）が実際に負かに直接
   依存する。したがって集計 ΔQ̄ の符号だけでなく **層別（chi/pon/kan・向聴・巡目・
   点況）が "どの局面で鳴けと薦めるか" の商品仕様に直結** → これまで判定非関与の
   exploratory 扱いだった層別が、判定後の商品設計では一級市民になる。
   **注意: これは DRCA の凍結解釈条件（`drca_probe_design.md` §5a/§4）を変更しない。**
   判定の解釈は凍結のまま。ここで言うのは "判定後の商品設計での利用価値" の話。
   frame1 の裾支配（負の裾=本物の手壊し / 正の裾に取りこぼし候補）が、この文脈では
   "鳴けと言うべき局面クラス" の設計材料になる。
4. **経済プリセットが第一級ロードマップに昇格。** 定数変更（議題2）は "鳴く AI を出す
   苦肉の策" ではなく商品の軸そのもの。特に **鳴き祝儀は現行報酬に不在のレバー**:
   現行チップは赤保持ベースで面前非依存（`reward_design_teacherfree.md` l.76–77 で確認、
   赤で和了れば鳴いても入る）、鳴きで失う門前価値=立直/裏/一発は素点側。
   したがって立直特化は祝儀構造のアーティファクトではなく本物の均衡（§1 と整合）。
   鳴き祝儀の追加は、測定済みの機会費用ギャップ（~5.5–5.9 チップ）を**直接補助する
   最も標的的な経済レバー**であり、議題2 の第一候補軸。

### 5.3 ヒアリングで新たに浮上した設計ギャップ（セッションの実装 TODO 候補・Gamba 質問ではない）

| # | ギャップ | 背景 |
|---|---|---|
| G1 | **live per-action 評価器** | 出力=「上位N手+評価値」には行動ごとの評価が要る。現アーキは actor π(行動分布) + critic V(s)(状態値・3ストリーム)のみで per-action Q は無い。DRCA の fork-by-replay は per-action だが live 助言には重すぎる。安価な実現手段（π 順位 + V、advantage/Q ヘッド増設、1手 lookahead 等）の設計が要る。DRCA を "商品の計測器" に格上げする用途 5.2-3 と同根 |
| G2 | **対人プレイ/助言レビュー ハーネス** | 強さ基準=人間実戦の合否を測る手段。既存の argmax 定性レビュー（Stage1 07-15 / Stage3 07-16）がその原型。牌譜 HTML ビューア（07-15 実装）は素材提示に使える。対局 or 助言レビューの回し方を定式化する必要 |
| G3 | **鳴き祝儀/面前祝儀を含む経済プリセットの config 化** | 議題2 の前提。現行 reward の chip 項（赤保持ベース・面前非依存）を、面前/鳴き条件付き祝儀を表現できる形に一般化する設計。1 branch = 1 variable で軸ごとに直列 |

これら G1–G3 は Gamba への追加ヒアリング不要の設計課題。DRCA 判定後の 0b セッションで
議題1（G2 が合否測定手段）・議題2（G3 が実験系前提）・商品化検討（G1）の各文脈に配分する。
